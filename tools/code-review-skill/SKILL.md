# Code Review Skill

Review code for bugs, security issues, and style violations.

## Instructions

When asked to review code:
1. Check for common bugs and logic errors
2. Look for security vulnerabilities (OWASP Top 10)
3. Verify error handling is complete
4. Check for code style consistency
5. Suggest improvements with explanations

## Output Format

For each issue found:
- **Severity**: Critical / Warning / Info
- **Location**: File and line reference
- **Issue**: Description of the problem
- **Fix**: Suggested resolution
